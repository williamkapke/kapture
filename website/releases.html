Version 1.2

Click anywhere

Kapture can now click and hover at a plain viewport coordinate — no selector required. When a page throws up an alert, the agent is no longer stuck: it sees the dialog and answers it. And a compose script can end with a screenshot or other read, so act-and-verify is one call.

Coordinate clicks reach inside iframes

Pass x and y (CSS viewport pixels) to click or hover instead of a selector. The events dispatch through the browser's input pipeline, so they hit whatever is at that point — including content inside iframes, which selectors have never been able to reach.

Dialogs no longer kill the run

An unexpected alert, confirm, or prompt freezes the page — and until now, every command after it just timed out while the assistant sat in the dark. Now the command that triggered the dialog returns immediately with its type and message, every other command fails fast with a DIALOG_OPEN error saying exactly what to do, and the new dialog tool answers it: OK or Cancel, with typed text for prompts. Unsaved-changes (onbeforeunload) dialogs work the same way.

compose can now end with a look

The most common sequence in the wild is act, then check: click and take a screenshot, fill and look for console errors, submit and read the DOM. A compose script may now end with one data-returning command — screenshot, dom, elements, elementsFromPoint, console_logs, or watch_console — so the whole act-and-verify round trip is a single call. A terminal screenshot even returns its image in the same response.

Version 1.1

Watch the network

Kapture can now see the network. Turn on monitoring for a tab, drive the page however you like, then read back every request it made — with response bodies on demand.

Network monitoring

Three new tools capture a tab's network traffic. Because the browser keeps no history, you turn monitoring on before the action you want to watch, then inspect what happened:

  • network_monitor — turn capture on or off for a tab. Watchers are tracked per client, so it stays on until everyone watching turns it off — and a client that disconnects is released automatically.
  • network_requests — list the captured requests (url, method, status, type, size, timing). Poll just the new ones by passing back the cursor from the previous call.
  • network_body — pull an individual request's bodies on demand, fetched live from the browser: the POST payload it sent and the response it got back.

Fixes & polish

  • Scroll inside any element. scroll with a selector and coordinates now scrolls within that element — inner panes, sidebars, infinite lists — and reports where it landed.
  • Input now works on background tabs. Clicks and keystrokes used to vanish silently when the target tab wasn't in front; the page is now told it's focused for the duration of the event, and responses include a warning when the tab is hidden.
  • Enter submits forms. keypress Enter now carries the character event browsers use for implicit form submission.
  • The DevTools panel no longer renders page content as HTML. Message data in the panel's log is now displayed as plain text, so markup coming back from a page can't distort the panel or inject content into it.
  • A quicker pointer. The visual mouse cursor glides to its target much faster and wears a violet glow so it's easy to spot.
Version 1.0

New ways to drive the page

This release adds the most-requested controls: running JavaScript, typing the way a person does, scrolling, and batching commands.

Run JavaScript in the page

The new evaluate tool runs JavaScript and returns the result, for the moments when no other tool quite fits. It stays off until you allow it: flip the red Allow JS execution switch in the toolbar popup or the DevTools panel. The grant is per tab and clears the instant that tab disconnects, so an assistant can never turn it on by itself.

Type like a real user

Three tools that go through the real keyboard pipeline, so they work on custom inputs and rich editors that ignore programmatic value-setting:

  • type — sends a string as individual keystrokes
  • insertText — drops in a whole block at once (great for large text and editors like Google Docs)
  • clear — empties a field reliably

Scroll

The new scroll tool brings an element into view, or scrolls to an exact coordinate.

Compose: many commands in one call

The new compose tool runs a sequence of commands against a tab in a single call, one command per line, stopping at the first failure. Fewer round-trips for multi-step actions like fill, then click, then scroll.
